# Daymon 실행 방법 (처음부터 순서대로)

## 1단계: 터미널 열기

- VS Code / Cursor에서 **터미널** 메뉴 → **새 터미널** (또는 `` Ctrl+` ``)
- 아래 명령어는 **프로젝트 폴더(`daymon`)가 열린 상태**에서 실행하세요.

---

## 2단계: 패키지 설치 (한 번만)

터미널에 입력 후 Enter:

```powershell
npm install
```

끝날 때까지 기다린 뒤, 에러 없이 끝나면 다음 단계로.

---

## 3단계: API 키 파일 만들기

### 3-1. `.env` 파일이 있는지 확인

- 프로젝트 루트에 `.env` 파일이 **없다**면 아래대로 만듭니다.

### 3-2. `.env` 파일 만들기 (Windows)

**방법 A – 복사해서 만들기**

1. 프로젝트에 `.env.example` 파일이 있음
2. 이 파일을 **복사**해서 이름을 **`.env`** 로 바꿉니다.
3. `.env` 파일을 연 다음, 아래 한 줄을 **본인 API 키**로 수정합니다.

```
ANTHROPIC_API_KEY=sk-ant-api03-여기에_본인_키_붙여넣기
```

**방법 B – 터미널로 복사 (PowerShell)**

```powershell
Copy-Item .env.example .env
```

그 다음 `.env` 파일을 열어서 `ANTHROPIC_API_KEY=sk-ant-...` 부분을 본인 키로 수정합니다.

### API 키 받는 곳

- https://console.anthropic.com/ 에서 가입/로그인 후 API 키 발급

---

## 4단계: 서버 두 개 실행하기

대화 기능은 **프론트(화면) + API 서버** 둘 다 켜져 있어야 합니다.

### 방법 A – 한 번에 실행 (추천)

터미널 **한 개**에서:

```powershell
npm run dev:all
```

- 그대로 두면 **API 서버**와 **웹(React)** 이 같이 실행됩니다.
- 터미널에 `Local: http://localhost:5173/` 같은 줄이 보이면 성공.

### 방법 B – 터미널 두 개로 실행

1. **첫 번째 터미널** (API 서버용)
   ```powershell
   npm run dev:server
   ```
   - `Daymon API 서버: http://localhost:3001` 이 보이면 그대로 둡니다.

2. **두 번째 터미널** (웹 화면용)  
   - **새 터미널**을 하나 더 연 다음:
   ```powershell
   npm run dev
   ```
   - `Local: http://localhost:5173/` 이 보이면 성공.

---

## 5단계: 브라우저에서 접속

1. 브라우저(Chrome, Edge 등)를 연다.
2. 주소창에 **아래 주소만** 입력합니다.  
   **`http://localhost:5173`**
3. Daymon 메인 화면이 보이면 **대화하기** 버튼을 누릅니다.
4. 메시지를 입력하고 전송해 봅니다.

---

## 자주 하는 실수

| 증상 | 원인 | 해결 |
|------|------|------|
| 페이지가 아예 안 뜸 | `npm run dev` 또는 `npm run dev:all` 을 안 함 | 4단계 다시 실행 |
| 대화하기 눌렀는데 "연결할 수 없습니다" | API 서버가 꺼져 있음 | `npm run dev:server` 또는 `npm run dev:all` 실행 |
| "ANTHROPIC_API_KEY가 설정되지 않았습니다" | `.env` 없거나 키가 잘못됨 | 3단계 다시 확인, `.env` 에 키 한 줄 넣기 |
| `localhost:5173` 말고 다른 주소로 열었음 (예: `file:///...`) | 그 주소에선 API 호출이 안 됨 | 반드시 **http://localhost:5173** 로 접속 |

---

## 정리

1. `npm install`
2. `.env` 만들고 `ANTHROPIC_API_KEY=본인키` 넣기
3. `npm run dev:all` (또는 터미널 2개로 `dev:server` + `dev`)
4. 브라우저에서 **http://localhost:5173** 접속 → **대화하기** 사용

여기까지 해도 안 되면, 터미널에 나온 **에러 메시지 전체**를 복사해서 알려주면 다음 단계 도와줄 수 있어요.
