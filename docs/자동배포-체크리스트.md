# Daymon 자동 배포 체크리스트

친구들에게 보여주려면 아래 순서대로 하면 됩니다.

---

## 1. GitHub 저장소 만들기

1. https://github.com 접속 → 로그인
2. 오른쪽 위 **+** → **New repository**
3. **Repository name**: `daymon` (또는 원하는 이름)
4. **Public** 선택
5. **Create repository** 클릭

---

## 2. 로컬에서 GitHub에 푸시

프로젝트 폴더(`daymon`)에서 터미널 열고:

```powershell
# 처음 한 번만
git init
git add .
git commit -m "Daymon 첫 배포"

# daymon-source / daymon 저장소 기준
git remote add origin https://github.com/daymon-source/daymon.git
git branch -M main
git push -u origin main
```

---

## 3. GitHub Pages 설정

1. GitHub에서 방금 만든 **daymon** 저장소 페이지로 이동
2. 위쪽 탭에서 **Settings** 클릭
3. 왼쪽 메뉴 맨 아래 **Pages** 클릭
4. **Build and deployment** 섹션에서:
   - **Source**: **GitHub Actions** 선택
5. 저장 (별도 버튼 없으면 이미 반영됨)

---

## 4. 자동 배포 확인

- **main** 브랜치에 `git push`할 때마다 자동으로 배포됩니다.
- 배포 상태: 저장소 **Actions** 탭에서 확인
- 2~3분 정도 지나면 배포 완료

---

## 5. 친구들에게 공유할 주소

배포가 끝나면:

**https://daymon-source.github.io/daymon/**

이 주소를 친구들에게 보내면 됩니다.

---

## 자주 하는 실수

| 증상 | 해결 |
|------|------|
| 404 Not Found | 주소 끝에 `/daymon/` 포함했는지 확인. Pages 설정에서 Source가 **GitHub Actions**인지 확인 |
| Actions에서 빌드 실패 | 저장소 **Actions** 탭에서 실패한 워크플로 클릭 → 로그 확인. `npm install` 또는 `npm run build` 에러가 많음 |
| push가 안 됨 | `git remote -v`로 주소 확인. `git push -u origin main` 다시 실행 |

---

## 코드 수정 후 다시 배포

수정한 뒤 배포만 다시 하고 싶으면:

```powershell
git add .
git commit -m "업데이트 내용"
git push
```

푸시하면 자동으로 다시 빌드·배포됩니다.
