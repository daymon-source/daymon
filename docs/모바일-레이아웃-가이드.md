# 모바일 레이아웃 일관성 가이드

모바일 기기마다 화면 크기·주소창·노치가 달라서 위치가 제각각인 문제는 **전세계 공통 이슈**입니다. 아래 방식으로 대응해 두었습니다.

---

## 1. **위치는 px 대신 %(비율) 사용**

- **문제**: `top: 320px`처럼 고정 px는 기기 높이에 따라 비율이 달라짐 (작은 폰에선 너무 아래, 큰 폰에선 위로 치우침).
- **해결**: `.monster-wrap--egg1`, `--egg2`, `--hatched`는 **`top: 44%`·`45%`** 처럼 **컨테이너 기준 %**로 배치.
- **효과**: 기기 크기와 관계없이 **같은 비율**에 알/몬스터가 오도록 유지.

---

## 2. **뷰포트 높이: 100vh 대신 100dvh**

- **문제**: 모바일에서 `100vh`는 **주소창을 포함한 전체 높이**라, 주소창이 접히면 실제 보이는 영역보다 커져서 스크롤/빈 공간이 생김.
- **해결**: `min-height: 100dvh`, `max-height: 100dvh` 사용. (`100vh`는 구형 브라우저용 fallback으로 유지.)
- **참고**: `svh`(작은 뷰포트), `lvh`(큰 뷰포트), `dvh`(동적) 중 **dvh**가 주소창 접힘/펼침에 맞춰 변하므로 일반적으로 권장.

---

## 3. **노치·홈 인디케이터: safe-area-inset**

- **문제**: 아이폰 등에서 화면 가장자리가 잘리거나 가려짐.
- **해결**: `index.html`에 **`viewport-fit=cover`** 설정, 하단 네비 등에는 **`padding-bottom: calc(0.5rem + env(safe-area-inset-bottom))`** 사용.
- **효과**: 기기별 안전 영역 안쪽으로만 콘텐츠가 들어가도록 유지.

---

## 4. **고정 위치(position: fixed) 주의**

- 모바일 브라우저는 `position: fixed`를 기기마다 다르게 해석하는 경우가 있음.
- **가능하면** `position: absolute` + 스크롤 영역 분리로 “고정처럼 보이게” 구성하는 편이 더 안정적. (현재 Daymon은 알 탭이 스크롤 없는 구조라 absolute 기준으로 배치.)

---

## 요약

| 이슈 | 대응 |
|------|------|
| 기기마다 알/몬스터 위치 다름 | `top`/`left`를 **%**로 지정 (px 사용 지양) |
| 주소창 때문에 높이/스크롤 어색함 | **100dvh** 사용 |
| 노치·홈 인디케이터에 가림 | **viewport-fit=cover** + **env(safe-area-inset-*)** |

새로 **위치를 넣는 요소**가 생기면 **px 대신 % 또는 rem**을 쓰고, 전체 높이가 필요하면 **dvh**를 사용하면 됩니다.
