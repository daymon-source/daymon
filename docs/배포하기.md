# Daymon 배포 가이드

다른 사람에게 보여주려면 **GitHub Pages**로 배포하면 됩니다.

---

## ⚠️ 중요: API 서버 문제

현재 Claude API 서버(`server/index.js`)는 **로컬에서만 작동**합니다.  
GitHub Pages는 **정적 파일만** 호스팅하므로 API 서버를 직접 실행할 수 없어요.

### 해결 방법

**옵션 1: 프론트엔드만 배포 (대화 기능 제외)**
- GitHub Pages로 배포
- 대화 기능은 작동하지 않지만, 몬스터 상호작용은 가능

**옵션 2: API 서버도 함께 배포**
- Vercel/Netlify/Railway 같은 서비스에 API 서버 배포
- 프론트엔드에서 API 주소를 변경

---

## 방법 1: GitHub Pages로 프론트엔드 배포

### 1단계: GitHub 저장소 만들기

1. https://github.com 에서 **새 저장소** 생성
   - 저장소 이름: `daymon` (또는 원하는 이름)
   - Public으로 설정
   - README는 추가하지 않아도 됨

### 2단계: 코드 푸시

터미널에서:

```powershell
# Git 초기화 (아직 안 했다면)
git init

# 모든 파일 추가
git add .

# 첫 커밋
git commit -m "Initial commit"

# GitHub 저장소 연결 (daymon-source/daymon)
git remote add origin https://github.com/daymon-source/daymon.git

# 푸시
git branch -M main
git push -u origin main
```

### 3단계: GitHub Pages 설정

1. GitHub 저장소 페이지에서 **Settings** 클릭
2. 왼쪽 메뉴에서 **Pages** 클릭
3. **Source**에서 **GitHub Actions** 선택
4. 저장

### 4단계: GitHub Actions 워크플로우 파일 만들기

프로젝트에 `.github/workflows/deploy.yml` 파일을 만들어야 해요.

---

## 방법 2: API 서버도 함께 배포 (Vercel)

### 1단계: Vercel에 API 서버 배포

1. https://vercel.com 에서 가입/로그인
2. **New Project** → GitHub 저장소 선택
3. **Root Directory**를 `server`로 설정
4. **Environment Variables**에 `ANTHROPIC_API_KEY` 추가
5. Deploy

### 2단계: 프론트엔드에서 API 주소 변경

`src/components/ChatModal.jsx`에서:

```javascript
const res = await fetch('https://your-api.vercel.app/api/chat', {
  // ...
})
```

## 빠른 배포 (GitHub Actions 자동화)

`.github/workflows/deploy.yml` 파일을 만들면 자동으로 배포됩니다.
